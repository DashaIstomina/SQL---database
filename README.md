# Проект работы с реляционной базой данных

В данном проекте представлены примеры решения SQL-задач с использованием:  
- группировки данных и агрегатных функций;  
- объединения таблиц (JOIN);  
- подзапросов;  
- общих табличных выражений (CTE);  
- оконных функций;  
- условных конструкций (CASE);  
- функций работы с датами и временем;  
- представлений (VIEW);  
- индексов (INDEX).  

# Описание таблиц базы данных  

**clients** - хранит информацию о клиентах банка, подающих заявки на получение кредита.  

client_id - уникальный идентификатор клиента  
last_name - фамилия клиента  
first_name - имя клиента  
middle_name - отчество клиента  
birth_date - дата рождения  
phone - номер телефона  
email - адрес электронной почты  
city - город проживания  
registration_date - дата регистрации клиента в системе  

**branches** - хранит информацию о филиалах банка.  

branch_id - уникальный идентификатор филиала  
name - название филиала  
city - город расположения филиала  
address - адрес филиала  

**employees** - хранит информацию о сотрудниках банка, обрабатывающих кредитные заявки.  

employee_id - уникальный идентификатор сотрудника  
branch_id - идентификатор филиала, в котором работает сотрудник  
last_name - фамилия сотрудника  
first_name - имя сотрудника  
position - должность сотрудника  
hire_date - дата приема на работу  

**loan_products** - хранит информацию о кредитных продуктах банка.  

loan_product_id - уникальный идентификатор кредитного продукта  
product_name - название кредитного продукта  
interest_rate - процентная ставка  
min_amount - минимальная сумма кредита  
max_amount - максимальная сумма кредита  
min_term_months - минимальный срок кредита (в месяцах)  
max_term_months - максимальный срок кредита (в месяцах)  

**loan_applications** - хранит информацию о заявках клиентов на получение кредита.  

application_id - уникальный идентификатор заявки  
client_id - идентификатор клиента  
loan_product_id - идентификатор кредитного продукта  
employee_id - идентификатор сотрудника, обработавшего заявку  
requested_amount - запрашиваемая сумма кредита  
requested_term_months - запрашиваемый срок кредита (в месяцах)  
application_date - дата подачи заявки  
current_status - текущий статус заявки  

**application_status_history** - хранит историю изменения статусов кредитных заявок.  

history_id - уникальный идентификатор записи истории  
application_id - идентификатор заявки  
status - статус заявки  
changed_at - дата и время изменения статуса  

**documents** - хранит справочник документов, необходимых для оформления кредита.  

document_id - уникальный идентификатор документа  
document_name - название документа  

**application_documents** - хранит информацию о документах, прикрепленных к кредитным заявкам.  

application_document_id - уникальный идентификатор записи  
application_id - идентификатор заявки  
document_id - идентификатор документа  
uploaded_at - дата и время загрузки документа  

**loans** - хранит информацию о выданных кредитах.  

loan_id - уникальный идентификатор кредита  
application_id - идентификатор заявки, по которой был выдан кредит  
issued_amount - сумма выданного кредита  
interest_rate - процентная ставка по кредиту  
issue_date - дата выдачи кредита  
closing_date - дата закрытия кредита  
loan_status - текущий статус кредита  

**payments** - хранит информацию о платежах по выданным кредитам.  

payment_id - уникальный идентификатор платежа  
loan_id - идентификатор кредита  
payment_date - дата платежа  
payment_amount - сумма платежа  
payment_status - статус платежа
